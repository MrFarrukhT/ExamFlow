```markdown
/improve_architecture

Analyze and suggest architectural improvements for a page, component, or function through collaborative dialogue.

Focus on architectural analysis and recommendations rather than implementation.

Prerequisites:
- Must run /analyze_architecture first
- Understand current behavior and contracts
- Identify specific improvement opportunities

Approach:
- Provide architectural suggestions and recommendations
- Engage in collaborative dialogue to explore options
- Discuss trade-offs and implications of different approaches
- Work together to design optimal architecture
- Focus on analysis rather than immediate implementation

Analysis Framework:
- Current Architecture Assessment: evaluate existing structure and patterns
- Behavioral Contracts: identify what functionality must remain identical
- API Boundaries: analyze input/output signatures and interfaces
- Side Effect Mapping: catalog all mutations, network calls, storage operations
- Performance Characteristics: assess current latency/memory constraints
- Scalability Concerns: identify potential bottlenecks and growth limitations

Architectural Discussion Areas:
- Component Decomposition: how to break down complex components
- State Management Strategy: optimal patterns for data flow and state handling
- Hook Extraction Opportunities: reusable logic extraction possibilities
- Separation of Concerns: UI vs business logic boundaries
- Error Handling Architecture: comprehensive error boundary strategies
- Performance Optimization: memoization and rendering optimization opportunities
- Type Safety Improvements: TypeScript enhancement strategies
- Code Organization: module structure and dependency management

Collaborative Output:
- Architecture Analysis: current state assessment with strengths/weaknesses
- Improvement Opportunities: prioritized list of potential enhancements
- Design Options: multiple architectural approaches with pros/cons
- Trade-off Discussion: performance vs maintainability vs complexity analysis
- Optimal Architecture Proposal: recommended structure after dialogue
- Implementation Roadmap: phased approach for gradual improvements
- Risk Assessment: potential issues and mitigation strategies
- Future Considerations: scalability and extensibility planning
- Decision Points: key choices that need stakeholder input
- Next Steps: recommended actions for moving forward

Dialogue Style:
- Ask clarifying questions about requirements and constraints
- Present multiple options with clear trade-offs
- Seek feedback on proposed approaches
- Iterate on designs based on discussion
- Validate understanding of priorities and goals
- Encourage exploration of alternative solutions
```
