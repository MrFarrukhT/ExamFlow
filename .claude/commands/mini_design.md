/mini_design

You will not write code.

Design the change in a concise, implementation-ready format.

Output:
- Interfaces: inputs/outputs (TS types if known), avoid hard-coded data
- Data Flow: short sequence or diagram-like list (how data moves through system)
- Error Handling: AppError kinds + when they’re raised
- Security/Perf Notes: where to validate/authz, expected p95, caching yes/no
- Observability: logs/metrics/traces to add (1–2 lines)
- Tests to Add: checklist (unit, integration, e2e happy/failure paths)
- Files: exact paths to touch/create
- Risks: top 1–2 things that could break and how to mitigate
